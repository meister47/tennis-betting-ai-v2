# Workflow: Tennis Betting AI

## 🎯 Цель скилла

Найти value-ставки на теннис, где коэффициенты букмекеров завышены относительно реальных шансов.

## 🔄 Полный цикл работы

### 1. **Сбор данных (Data Collection)**
```
Источники → Парсинг → Нормализация → Хранение
```

**Источники:**
- ATP Tour / WTA — расписание, результаты, статистика
- Flashscore / ESPN — live scores, подробности матчей
- Букмекерские сайты — коэффициенты
- Tennis Abstract — расширенная статистика

**Частота:** Ежедневно в 08:00 и 18:00 МСК

### 2. **Анализ матча (Match Analysis)**
```
Статистика → Контекст → Вероятность → Value
```

**Параметры анализа:**
- Рейтинг игрока (ATP/WTA)
- Статистика на текущем покрытии
- Head-to-head история
- Текущая форма (последние 5 матчей)
- Травмы и пропуски
- Психологические факторы (титулы, давление)

**Формула оценки вероятности:**
```
P(победа) = 0.4*Рейтинг + 0.3*Форма + 0.2*H2H + 0.1*Контекст
```

### 3. **Оценка value (Value Assessment)**
```
Коэффициенты → Подразумеваемая вероятность → Edge
```

**Протокол Clean Ticket:**
1. Получить коэффициенты букмекера
2. Рассчитать подразумеваемую вероятность
3. Сравнить с нашей оценкой вероятности
4. Рассчитать edge (преимущество)

**Формула edge:**
```
Edge = (Наша_вероятность * Коэффициент) - 1
```

**Критерии ставки:**
- Edge > 0.05 (5%) — сильная ставка
- Edge > 0.02 (2%) — умеренная ставка  
- Edge <= 0 — не ставить

### 4. **Risk Management**
```
Размер банкролла → Критерий Келли → Размер ставки
```

**Критерий Келли:**
```
Размер_ставки = (Edge / (Коэффициент - 1)) * Банкролл
```

**Ограничения:**
- Максимум 2% от банкролла на одну ставку
- Максимум 5% от банкролла в день
- Stop-loss: -20% от банкролла

### 5. **Рекомендация (Recommendation)**
```
Анализ → Обоснование → Рекомендация → Размер
```

**Формат рекомендации:**
```json
{
  "match": "Alcaraz vs Sinner",
  "tournament": "Monte Carlo Masters",
  "date": "2026-04-11",
  "recommendation": "BET",
  "selection": "Alcaraz",
  "odds": 1.60,
  "stake": 150,
  "edge": 0.08,
  "confidence": 0.75,
  "reasoning": "Alcaraz имеет лучше статистику на грунте...",
  "alternatives": ["Тотал >21.5", "Alcaraz -3.5"]
}
```

### 6. **Отслеживание (Tracking)**
```
Ставка → Результат → Анализ → Обучение
```

**Метрики:**
- ROI (Return on Investment)
- Hit Rate (процент выигранных ставок)
- Average Odds
- Profit/Loss

**Обучение модели:**
- Корректировка весов параметров
- Учёт новых факторов
- Адаптация к изменениям

## 📅 Ежедневный workflow

### Утро (08:00 МСК)
1. Проверить расписание на день
2. Собрать статистику по игрокам
3. Найти коэффициенты
4. Проанализировать матчи
5. Отправить рекомендации

### Вечер (18:00 МСК)  
1. Проверить результаты
2. Обновить статистику
3. Скорректировать модель
4. Подготовить отчёт

### Ночь (23:00 МСК)
1. Резервное копирование данных
2. Оптимизация базы данных
3. Проверка ошибок

## 🚀 Быстрый старт для разработчика

1. Клонировать репозиторий
2. Установить зависимости: `npm install`
3. Настроить конфигурацию: `cp config.example.json config.json`
4. Запустить тесты: `npm test`
5. Запустить анализ: `node scripts/analyze.js`
