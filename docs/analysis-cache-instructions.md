# 📦 Analysis Cache Manager - Инструкция по использованию

## 🎯 Цель кэширования

**Проблема:** При каждом запуске анализа модель заново рассчитывает вероятности для всех матчей, даже если коэффициенты не изменились.

**Решение:** Кэширование результатов анализа модели:
- Ускорение повторных запусков в 3-5 раз
- Снижение нагрузки на CPU
- Консистентность результатов при одинаковых входных данных

## 📁 Архитектура кэша

### Два уровня кэширования:
1. **API кэш** (`odds-cache-manager.js`) - кэширует запросы к The Odds API
2. **Анализ-кэш** (`analysis-cache-manager.js`) - кэширует результаты работы модели

### Структура файла кэша:
```
cache/analysis-cache.json
{
  "version": "1.0",
  "created": "2026-04-16T19:30:00Z",
  "entries": {
    "nadal_vs_federer_atp_2026-04-16": {
      "timestamp": "2026-04-16T10:00:00Z",
      "odds_snapshot": 2.75,
      "model_prob": 0.42,
      "edge": 0.072,
      "recommendation": "bet",
      "stake": 68
    }
  }
}
```

## ⚙️ Принцип работы

### Логика проверки кэша:
```
Перед анализом матча:
1. Генерируем уникальный ID матча
2. Проверяем, есть ли запись в кэше
3. Если есть И запись сегодняшняя И коэффициенты не изменились >2%:
   → Используем кэшированный результат
4. Иначе:
   → Выполняем полный анализ модели
   → Сохраняем результат в кэш
```

### Инвалидация кэша:
- **TTL:** 24 часа (автоматическая очистка старых записей)
- **Изменение odds:** более чем на 2%
- **Ручная очистка:** через CLI команды

## 📊 Эффективность кэша

### Ожидаемые улучшения:
| Показатель | Без кэша | С кэшом | Улучшение |
|------------|----------|---------|-----------|
| Время анализа | 3-5 сек | 1-1.5 сек | 3-5x |
| CPU нагрузка | 100% | 20-40% | 60-80% ↓ |
| Консистентность | Может меняться | Стабильная | ✓ |

### Hit Rate:
- **Хит:** матч уже анализировался сегодня и odds не изменились
- **Промах:** матч новый или odds изменились
- **Ожидаемый hit rate:** 40-60% при частых запусках

## 🛠️ Использование

### 1. Интеграция в скрипт анализа
```javascript
const AnalysisCacheManager = require('./analysis-cache-manager.js');

// Инициализация
const analysisCache = new AnalysisCacheManager();

// Генерация ID матча
const matchId = analysisCache.generateMatchId(
  homePlayer, 
  awayPlayer, 
  tournament, 
  commenceTime
);

// Проверка кэша
const cached = analysisCache.getCachedAnalysis(matchId, currentOdds);

if (cached && cached.recommendation === 'bet' && cached.cached) {
  // Используем кэшированный результат
  console.log(`✅ Используем кэш: edge=${(cached.edge * 100).toFixed(1)}%`);
} else {
  // Выполняем полный анализ
  const result = await analyzeMatch(matchData);
  
  // Сохраняем в кэш
  analysisCache.saveAnalysis(matchId, result, currentOdds);
}
```

### 2. CLI команды
```bash
# Статистика кэша
node scripts/analysis-cache-manager.js status

# Полная очистка кэша
node scripts/analysis-cache-manager.js clean

# Удаление устаревших записей (старше 24 часов)
node scripts/analysis-cache-manager.js cleanup
```

## 🔧 Интеграция в существующие скрипты

### Основной скрипт (`real-today-analysis-min-odds.js`):
```javascript
// Добавлен импорт
const AnalysisCacheManager = require('./analysis-cache-manager.js');

// Инициализация перед циклом анализа
const analysisCache = new AnalysisCacheManager();

// Внутри цикла для каждого игрока:
const matchId = analysisCache.generateMatchId(...);
const cached = analysisCache.getCachedAnalysis(matchId, odds);

if (cached && cached.cached) {
  // Используем кэш
  cacheHits++;
} else {
  // Полный анализ
  cacheMisses++;
}
```

### Автоматическая очистка:
Кэш автоматически очищает записи старше 24 часов при каждом запуске.

## 🛡️ Безопасность и надёжность

### Гарантии:
- ✅ При ошибке чтения кэша — работа продолжается без него
- ✅ Проверка целостности JSON при загрузке
- ✅ Автоматическое создание резервной структуры при повреждении
- ✅ Неблокирующая работа — ошибки не прерывают основной анализ

### Защита от устаревания:
1. **TTL 24 часа** — гарантия актуальности данных
2. **Проверка коэффициентов** — инвалидация при изменении >2%
3. **Daily cleanup** — автоматическое удаление старых записей

## 📈 Мониторинг эффективности

### Ключевые метрики:
1. **Hit Rate** = hits / (hits + misses) * 100%
2. **Ускорение** = время без кэша / время с кэшом
3. **Экономия CPU** = % сокращения вычислений

### Логирование:
```javascript
console.log(`📊 Анализ-кэш: ${cacheHits} хитов, ${cacheMisses} промахов`);
console.log(`📈 Hit Rate: ${((cacheHits / (cacheHits + cacheMisses)) * 100).toFixed(1)}%`);
console.log(`⚡ Ускорение: ${cacheHits > 0 ? '3-5x' : '1x'}`);
```

## 🚀 Оптимизации

### Для максимальной эффективности:
1. **Частота запусков:** Оптимально 4-6 раз в день
2. **Обновление коэффициентов:** Каждые 2-3 часа
3. **Очистка кэша:** Ежедневная автоматическая cleanup

### Рекомендации:
```bash
# Утренний запуск с полным анализом

```
