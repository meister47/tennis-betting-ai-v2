# 🎾 Система команд для ставок

## 🦊 НОВАЯ СИСТЕМА: Обучение на ошибках

Теперь система анализирует проигравшие ставки, извлекает уроки и проверяет новые ставки по выученным правилам.

## 📋 Основные команды:

### 1. **Добавить новую ставку (с проверкой правил)**
```bash
./scripts/add-bet-with-check.sh "[событие]" "[рынок]" [коэффициент] [сумма]
```
**Пример:**
```bash
./scripts/add-bet-with-check.sh "Синнер vs Рууд" "Синнер победит" 1.50 50
```

### 2. **Проверить ставку по правилам**
```bash
python3 scripts/check-bet-rules.py "[событие]" "[рынок]" [коэффициент]
```
**Пример:**
```bash
python3 scripts/check-bet-rules.py "Зверев vs Синнер" "Зверев +1.5 сета" 1.80
```

### 2.1 **ML рекомендация для ставки**
```bash
./scripts/ml-bet-check.sh "[событие]" "[рынок]" [коэффициент]
```
**Пример:**
```bash
./scripts/ml-bet-check.sh "Синнер vs Алькарас" "Синнер победит" 1.80
```

### 2.2 **Быстрая ML проверка**
```
"ML прогноз: [игрок] vs [оппонент] @ [коэффициент]"
```
**Пример:**
```
"ML прогноз: Синнер vs Алькарас @ 1.80"
```

### 3. **ML анализ ставки**
```
"Анализ ставки #[номер]"
```
**Пример:**
```
"Анализ ставки #3"
```

### 4. **Обновить результат ставки**
```
"Ставка #[номер]: [результат]"
```
**Результаты:** `выигрыш`, `проигрыш`, `возврат`
**Примеры:**
```
"Ставка #1: выигрыш"
"Ставка #2: проигрыш"
"Ставка #3: возврат"
```

### 4. **Посмотреть статистику**
```
"Статистика ставок"
```

### 5. **Посмотреть ожидающие ставки**
```
"Ожидающие ставки"
```

### 6. **Посмотреть выученные уроки**
```
"Уроки ставок"
```

### 7. **Проверить результаты**
```
"Проверить результаты"
```

### 8. **Добавить заметку к ставке**
```
"Заметка к ставке #[номер]: [текст]"
```

## 📝 Формат данных в базе:
Каждая ставка имеет:
- `id` - номер ставки
- `date` - дата добавления
- `event` - событие
- `market` - рынок
- `odds` - коэффициент
- `stake` - сумма ставки
- `result` - результат (pending/won/lost/void)
- `return` - возврат
- `profit` - прибыль
- `status` - статус
- `notes` - заметки

## 🚦 Текущие правила (на основе 2 проигравших ставок):

### 🚫 ИЗБЕГАТЬ:
1. **Ставки против игрока в пиковой форме** (Синнер, Алькарас)
   - Урок из ставки #1: Зверев +1.5 сета vs Синнер
   - Правило: Не ставить против игрока, выигравшего последние 3+ турнира

2. **Точный тотал 3 сета в матчах доминирующих игроков**
   - Урок из ставки #4: Точный тотал сетов: 3 (Алькарас vs Синнер)
   - Правило: Оба игрока >70% побед в 2 сетах → избегать ставок на 3 сета

3. **Гандикапы против топ-игроков с коэффициентом <2.00**
   - Урок из ставки #1
   - Правило: Для андердогов против топ-5 нужен коэффициент >2.50

### ✅ РЕКОМЕНДУЕТСЯ:
1. Ставки НА игроков в пиковой форме
2. Победитель матча вместо точных счетов
3. Коэффициенты >2.50 для андердогов
4. Проверка статистики на конкретной поверхности
5. Анализ истории личных встреч

## 📁 Файлы системы:
- `bets-db.json` - база данных ставок
- `betting-rules.yaml` - правила на основе уроков
- `lessons-learned.md` - анализ проигравших ставок
- `betting-analysis-system.md` - система анализа
- `scripts/check-bet-rules.py` - скрипт проверки
- `scripts/add-bet-with-check.sh` - скрипт добавления с проверкой

## 🔄 Как система учится:
1. Проигрыш ставки → детальный анализ ошибок
2. Добавление урока в `lessons-learned.md`
3. Обновление правил в `betting-rules.yaml`
4. Проверка будущих ставок по новым правилам
5. Улучшение стратегии с каждой ошибкой

*Система становится умнее с каждой проигравшей ставкой!*
