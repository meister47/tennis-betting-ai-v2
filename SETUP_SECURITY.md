# 🔐 Настройка безопасности системы теннисных ставок

Этот файл поможет вам безопасно настроить API ключи для системы.

---

## 📋 ПРЕДВАРИТЕЛЬНЫЕ ШАГИ

### 1. Получите ключ The Odds API
- Перейдите на https://the-odds-api.com/
- Зарегистрируйтесь (бесплатно)
- На странице Dashboard получите ваш API ключ
- Бесплатный план: 50 запросов/месяц

### 2. Подготовьте среду
Убедитесь, что у вас установлен Node.js и npm:
```bash
node --version  # Должно быть 16+
npm --version   # Должно быть 8+
```

---

## 🛠️ НАСТРОЙКА

### Шаг 1: Создайте файл окружения
```bash
# Скопируйте шаблон
cp .env.example .env.local

# Отредактируйте файл (замените your_api_key_here)
nano .env.local
# или
vim .env.local
# или
code .env.local
```

### Шаг 2: Добавьте ваш ключ
В файле `.env.local` замените:
```bash
THE_ODDS_API_KEY=your_api_key_here_1234567890abcdef
```
на ваш реальный ключ, например:
```bash
THE_ODDS_API_KEY=5bade59990c62f13daecce0427ec665e
```

### Шаг 3: Убедитесь, что файл защищён
```bash
# Проверьте, что .env.local в .gitignore
grep ".env.local" .gitignore

# Если нет, добавьте
echo ".env.local" >> .gitignore
echo ".env.*.local" >> .gitignore
```

## 🔧 ЗАПУСК СИСТЕМЫ

### Вариант А: Ручной запуск (разработка)
```bash
# Загрузите переменные окружения
export $(grep -v '^#' .env.local | xargs)

# Запустите систему
node scripts/analyze-with-real-odds.js
```

### Вариант Б: Использование dotenv (рекомендуется)
Система уже настроена на автоматическую загрузку переменных из `.env.local`.

Просто запустите:
```bash

```

## ⚠️ ЧТО ДЕЛАТЬ, ЕСЛИ КЛЮЧ НЕ РАБОТАЕТ

### Признаки проблем:
1. Ошибка "Invalid API key"
2. Ошибка "Rate limit exceeded"
3. Пустые результаты

### Решения:
1. **Проверьте ключ:** Убедитесь, что ключ скопирован полностью
2. **Обновите ключ:** Получите новый ключ на сайте
3. **Проверьте лимиты:** Бесплатный план - 50 запросов/месяц

---

## 🔒 БЕЗОПАСНОСТЬ

### ❌ ЧЕГО НЕ ДЕЛАТЬ:
1. **НЕ коммитьте** `.env.local` в Git
2. **НЕ делитесь** вашим ключом публично
3. **НЕ используйте** один ключ на нескольких серверах

### ✅ ЧТО ДЕЛАТЬ:
1. **Используйте** разные ключи для разработки и продакшена
2. **Регулярно обновляйте** ключи (раз в 3-6 месяцев)
3. **Отзывайте** ключи при подозрении на компрометацию

## 📝 ПРИМЕРЫ ИСПОЛЬЗОВАНИЯ

### В коде теперь используется:
```javascript
// Вместо хардкоженного ключа:
// const API_KEY = '5bade59990c62f13daecce0427ec665e'; // ❌ СТАРОЕ

// Используется переменная окружения:
const API_KEY = process.env.THE_ODDS_API_KEY;
if (!API_KEY) {
  console.error('❌ THE_ODDS_API_KEY не установлен');
  console.error('   Создайте .env.local с ключом');
  process.exit(1);
}
```

### Для тестирования:
```bash
# Можно временно установить переменную
export THE_ODDS_API_KEY=your_key_here
node scripts/analyze-with-real-odds.js
```

## 🔄 МИГРАЦИЯ С ХАРДКОДИРОВАННЫХ КЛЮЧЕЙ

Если вы обновляетесь со старой версии, где ключи были в коде:

1. **Отзовите старый ключ** в The Odds API
2. **Создайте новый ключ**
3. **Добавьте его в `.env.local`**
4. **Удалите старые ключи из кода** (уже сделано)
